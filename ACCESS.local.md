# Web App API: баланс, оплата и ссылка Remnawave

## Почему после оплаты нет ссылки на подключение

**Успешная оплата (`/api/topup/create` → postback → зачисление)** только пополняет **баланс** пользователя в БД. Панель Remnawave и **subscription URL** создаются в другой операции — при **покупке тарифа**, когда с баланса списывается цена плана и в Remnawave создаётся пользователь VPN.

В боте то же самое: после пополнения приходит «Оплата подтверждена» и кнопка **«Активировать VPN»** — это как раз переход к покупке/активации, а не автовыдача ссылки сразу после платежа.

## Два шага для веб-воронки

1. **POST `/api/topup/create`** — счёт и оплата. В ответе сохраните:
   - `data.orderId` — для опроса статуса;
   - **`data.billingTelegramId`** и **`data.userId`** — чтобы вызвать покупку после успеха (для лидов `billingTelegramId` выглядит как `lead:тип:КОД`).

2. Дождаться успеха (postback / **`GET /api/topup/:orderId/status`** → `SUCCESS`, `credited: true`).

3. **POST `/api/subscription/buy`** с телом:
   - либо `{ "telegramId": "<реальный id или billingTelegramId>", "planId": "M1" }`,
   - либо `{ "lead_type": "...", "lead_code": ".......", "planId": "M1" }` (лид должен уже существовать после шага 1).

В ответе `subscription.subscriptionUrl` / `subscriptionUrl2` — это и есть ссылки для клиента; `remnawaveUuid` хранится в БД у подписки.

## Важно

- Один только успешный платёж **не** создаёт подписку и **не** вызывает Remnawave — это сделано намеренно (разделение «пополнение» и «покупка плана»).
- Если Remnawave не настроен (`REMNAWAVE_*`), в коде может подставляться заглушка или пустой URL — смотрите логи `[Remnawave]` на сервере.
